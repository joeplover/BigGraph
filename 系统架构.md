# 微服务整合架构

## 1. 总体结构

```text
Vue3 Frontend
    |
    v
FastAPI Gateway
    |
    +-- Zookeeper Service Registry
    |
    +-- RAG Service
    |
    +-- PPT Agent Service
    |
    +-- GPT Workflow Service
```

## 2. 服务边界

### RAG Service

职责：

- 知识库管理
- 文档上传
- 文档解析与向量入库
- RAG 检索问答
- 引用来源和 trace 返回

网关前缀：

```text
/api/rag/*
```

### PPT Agent Service

职责：

- 创建 PPT 制作会话
- 上传文字资料
- 聊天式需求补全
- 生成 PPT 大纲与 PPTX
- 下载 PPTX

网关前缀：

```text
/api/ppt/*
```

### GPT Workflow Service

职责：

- 图像项目管理
- 剧本分镜
- 角色卡和场景卡
- 文生图
- 图生图
- 本地生成结果保存

网关前缀：

```text
/api/workflow/*
```

静态资源代理：

```text
/workflow-static/*
```

## 3. Zookeeper 路径设计

根路径：

```text
/enterprise-rag/services
```

服务节点：

```text
/enterprise-rag/services/rag-service
/enterprise-rag/services/ppt-agent-service
/enterprise-rag/services/gpt-workflow-service
```

节点内容：

```json
{
  "name": "rag-service",
  "base_url": "http://127.0.0.1:8001",
  "health_path": "/health",
  "capabilities": ["knowledge-base", "rag-chat"]
}
```

## 4. 网关行为

网关启动时：

```text
1. 读取 .env 配置
2. 连接 Zookeeper
3. 创建 /enterprise-rag/services 路径
4. 写入或更新三套服务节点
5. 暴露 /api/services 查询服务目录
6. 按服务名解析目标地址并代理请求
```

当 Zookeeper 不可用时：

```text
1. 不阻断网关启动
2. 使用 .env 中的静态服务地址
3. /api/services 仍返回三套服务信息
```

## 5. 前端适配

Vue3 前端提供三个工作区：

```text
1. RAG 知识库
   - 创建知识库
   - 上传文档
   - 提问与引用展示

2. PPT Agent
   - 新建会话
   - 上传资料
   - 聊天推进 PPT 制作
   - 下载 PPTX

3. 图像工作流
   - 创建项目
   - 刷新项目列表
   - 输入提示词生成图片
   - 展示生成结果
```

## 6. 扩展方向

后续建议：

```text
1. 每个服务内部原生注册 Zookeeper ephemeral node
2. 网关增加服务健康检查和熔断
3. 增加统一认证与用户上下文透传
4. 增加统一任务中心
5. 增加服务级 OpenAPI 聚合
6. 将前端构建产物挂载到 Gateway 或 Nginx
```
